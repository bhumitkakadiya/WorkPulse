PHASE 7: PERFORMANCE, ACCESSIBILITY & FINAL QA    Loading states, error handling, responsiveness, and accessibility.  7.1 — Loading StatesEvery page that fetches data from the API must show the SkeletonLoader component (created in Phase 1.5) while the API call is pending. No page should show a blank white/dark screen during loading.Every button that triggers an API call must show a loading spinner inside it while the call is pending and be disabled to prevent double-submission.Charts must show a skeleton placeholder (gray animated rectangle) while data is loading.7.2 — Error HandlingEvery API call must have a try/catch. On error, use the Toast system (Phase 1.9) to show an error toast.If the entire page data fails to load (e.g. network error), show a full-page error state with: an error icon, 'Something went wrong' title, the error message in muted text, a 'Retry' button that re-fetches the data.401 Unauthorized errors must redirect to the login page automatically.Form submissions that fail must show inline API error messages below the relevant field, not just a generic toast.7.3 — ResponsivenessSidebar: on screens below 768px width, the sidebar must be hidden by default and toggled by a hamburger menu button in the topbar.
